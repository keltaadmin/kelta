## Versioning

KELTA uses Semantic Versioning.

```
MAJOR.MINOR.PATCH
```

Example:

```
1.0.0
```

where:

- **MAJOR** — incompatible API changes.
- **MINOR** — new backward-compatible functionality.
- **PATCH** — backward-compatible bug fixes.

---

## Release Types

### Alpha

Early development.

Example:

```
0.9.0-alpha1
```

### Beta

Feature complete but still under testing.

Example:

```
0.9.0-beta1
```

### Release Candidate

Production candidate.

Example:

```
1.0.0-rc1
```

### Stable

Production release.

Example:

```
1.0.0
```
